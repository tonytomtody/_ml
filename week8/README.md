# 非 Transformer 語言模型 | Character-Level Language Model (Non-Transformer)

## 📋 概述 (Overview)

這是一個**純 NumPy 實現**的字符級語言模型，完全**不依賴深度學習框架**（如 TensorFlow），特別為 **Windows ARM 環境**優化設計。

程式包含兩種不同的語言模型方案：
- **N-gram 統計模型**：快速、無需訓練
- **神經網路模型**：更強大但需要訓練時間

### 🎯 主要特性
- ✅ 純 NumPy 實現（只依賴 NumPy）
- ✅ 不需要 TensorFlow/PyTorch（Windows ARM 兼容）
- ✅ 支援中文字符
- ✅ 互動式使用者界面
- ✅ 可保存生成結果
- ✅ 兩種演算法可選

### 類別設計

#### 1. **SimpleNeuralNetwork** (簡單神經網路)
純 NumPy 實現的前向神經網路
```
輸入層 → 隱藏層(ReLU) → 輸出層(Softmax)
```

**主要方法：**
- `forward(X)` - 前向傳播
- `backward(X, y, output)` - 反向傳播（梯度下降）
- `softmax(x)` - Softmax 激活函數
- `train(X, y, epochs)` - 訓練循環

#### 2. **CharLevelLanguageModel** (字符級語言模型)
基於神經網路的字符級預測模型

**主要方法：**
- `build_vocab()` - 構建字符詞彙表
- `encode_text()` - 文本編碼為數字
- `generate_sequences()` - 生成訓練序列
- `train(epochs)` - 訓練模型
- `generate_text(seed_text, num_chars)` - 文本生成

#### 3. **NGramLanguageModel** (N-gram 模型)
基於統計的字符級預測模型

**主要方法：**
- `train()` - 統計 N-gram 頻率
- `generate_text(seed_text, num_chars)` - 文本生成

## 🧠 演算法原理 (Algorithm Principles)

### 方案 1: N-gram 統計模型

#### 原理
N-gram 模型基於**馬爾可夫鏈**的思想：下一個字符只取決於前 N-1 個字符的**條件概率**。

#### 數學方程
```
P(c_i | c_{i-n+1}, ..., c_{i-1}) = Count(c_{i-n+1}, ..., c_i) / Count(c_{i-n+1}, ..., c_{i-1})
```

#### 工作流程

**1. 訓練階段**
```
文本: "小貓在地上 小狗在路上"

4-gram 統計（n=4）:
上下文「小貓在」 → 出現「地」1次
上下文「貓在地」 → 出現「上」1次
上下文「在地上」 → 出現「 」1次
...
```

**2. 生成階段**
```python
種子文本 = "小貓"
上下文 = "小貓" (最後3個字)
如果上下文在統計表中:
    next_char = 頻率最高的字符
否則:
    next_char = 隨機選擇
```

#### 優勢與限制
| 特性 | N-gram |
|------|--------|
| **速度** | ⚡ 非常快 (毫秒級) |
| **記憶體** | 💾 低 |
| **訓練** | 🚀 無需訓練 |
| **效果** | 📊 中等（受文本大小影響） |
| **上下文** | 🔍 短期依賴 |

#### 範例
```
訓練文本: "小貓吃魚 小狗吃肉"

3-gram 模型:
「小貓吃」→「魚」(1次)
「貓吃魚」→「 」(1次)
「吃魚 」→「小」(1次)

生成: 種子"小貓" → "小貓吃魚小..."
```

### 方案 2: 神經網路模型

#### 原理
使用**淺層前向神經網路**進行字符預測。與 Transformer 不同，這是一個簡單的多層感知機 (MLP)。

#### 架構
```
輸入層 (seq_length × vocab_size)
    ↓ [Dense + ReLU]
隱藏層 (256個單元)
    ↓ [Dense + Softmax]
輸出層 (vocab_size)
    ↓ 
預測下一個字符的概率分佈
```

#### 工作流程

**1. 預處理**
```
原始文本: "小貓坐在桌上"
字符詞彙表: {'小':0, '貓':1, '坐':2, '在':3, ...}

One-hot 編碼示例:
'小' → [1, 0, 0, 0, ...]
'貓' → [0, 1, 0, 0, ...]
```

**2. 序列生成**
```
seq_length = 10

序列 1: [小,貓,坐,在,桌,上,...] → 預測下一個字
序列 2: [貓,坐,在,桌,上,...] → 預測下一個字
...
```

**3. 訓練**
```
前向傳播:
X(批次, seq_length×vocab)
  ↓ W1 + b1
Z1 = X·W1 + b1
  ↓ ReLU
A1 = ReLU(Z1)
  ↓ W2 + b2
Z2 = A1·W2 + b2
  ↓ Softmax
A2 = Softmax(Z2)

損失函數: Cross-Entropy Loss
L = -Σ y_i·log(ŷ_i)

反向傳播: 計算梯度並更新權重
W1 ← W1 - α·dW1
b1 ← b1 - α·db1
W2 ← W2 - α·dW2
b2 ← b2 - α·db2
```

**4. 文本生成**
```python
種子文本 = "小貓"
for 每個時步:
    將種子文本編碼為 one-hot
    前向傳播得到概率分佈
    使用"溫度採樣"選擇下一個字符
    更新種子文本
```

#### 溫度採樣 (Temperature Sampling)
增加生成文本的多樣性：
```python
T = 0.7  # 溫度參數
P'_i = exp(log(P_i) / T) / Σ exp(log(P_j) / T)
```
- T < 1.0：結果更確定（重複性強）
- T = 1.0：原始概率分佈
- T > 1.0：結果更隨機（多樣化強）

#### 優勢與限制
| 特性 | 神經網路 |
|------|--------|
| **效果** | 📈 更好（學習複雜模式） |
| **記憶體** | 💾 中等 |
| **訓練時間** | ⏱️ 數分鐘 |
| **速度** | 🚀 中等 |
| **上下文** | 🔍 可學習較長依賴 |

#### 數據流範例
```
文本: "小貓吃魚"
seq_length = 2

訓練樣本:
輸入: "小貓" → 輸出: "吃" (編碼為字符索引)
輸入: "貓吃" → 輸出: "魚"
...

訓練後可生成:
種子"小貓" → 預測"吃" → 預測"魚" → ...
```

## 📊 技術細節 (Technical Details)

### 激活函數

#### ReLU (整流線性單元)
```python
def relu(x):
    return max(0, x)
```
特點：
- 非線性：增加模型表達力
- 計算快：O(1) 時間複雜度
- 缺點：無法負值輸出（死亡 ReLU）

#### Softmax
```python
def softmax(x):
    e_x = exp(x - max(x))
    return e_x / sum(e_x)
```
特點：
- 輸出概率分佈（求和為 1）
- 可微分：適合反向傳播
- 用於多分類問題

### 損失函數

**交叉熵損失 (Cross-Entropy)**
```
L = -Σ y_true_i · log(y_pred_i)
```

### 梯度下降優化

```python
# 簡單梯度下降
W ← W - α·∇L(W)
b ← b - α·∇L(b)
```

其中 α 是學習率（預設 0.01）

### 複雜度分析

| 操作 | 時間複雜度 | 空間複雜度 |
|------|-----------|-----------|
| N-gram 訓練 | O(n) | O(nk) |
| N-gram 生成 | O(m) | O(1) |
| 神經網路前向 | O(ijk) | O(ij+jk) |
| 神經網路反向 | O(ijk) | O(ij+jk) |

其中 i = 輸入維度，j = 隱藏維度，k = 輸出維度

## 🐛 常見問題 (FAQ)

### Q1: 為什麼不用 Transformer？
**A:** Transformer 需要 TensorFlow/PyTorch，在 Windows ARM 上有相容性問題。本方案使用 N-gram 和簡單神經網路，不依賴任何深度學習框架。

### Q2: 生成的文本品質不好？
**A:** 
- 增加訓練數據
- 增加隱藏層大小
- 增加訓練 epoch
- 嘗試不同的溫度採樣值

### Q3: 訓練太慢？
**A:** 使用 N-gram 替代。或減少 `seq_length` 和隱藏層大小。

### Q4: 如何處理英文？
**A:** 程式已支援英文。修改 `tw.txt` 為英文文本即可。

### Q5: 可以保存模型嗎？
**A:** 可以使用 pickle 序列化模型，或修改程式自動保存權重。

## 📝 理論背景 (Theoretical Background)

### 語言建模基礎
語言模型的目標是計算句子的概率：
```
P(w_1, w_2, ..., w_n) = P(w_1) · P(w_2|w_1) · P(w_3|w_1,w_2) · ...
```

在字符級別：
```
P(text) = ∏ P(char_i | char_1, ..., char_{i-1})
```

### N-gram 的馬爾可夫假設
簡化为條件獨立性：
```
P(char_i | char_1, ..., char_{i-1}) ≈ P(char_i | char_{i-n+1}, ..., char_{i-1})
```

### 神經網路的優勢
學習對應函數：
```
f(X) = softmax(W2 · relu(W1 · X + b1) + b2)
可以捕捉 N-gram 無法表達的複雜模式
```
